❓ How to Use

🎙️ Tab 1 — Transcribe Audio / Video

  1. Upload — drag & drop an audio file or MP4 video onto the upload area, or click it to browse. Supported formats: WAV, MP3, M4A, FLAC, OGG, WebM, MP4 (up to 1 GB). MP4 files are transcribed by automatically extracting their audio track.
  2. Speaker Identification — enable the toggle to identify distinct voices in the audio. Each speaker is labelled SPEAKER_00, SPEAKER_01, etc. with timestamps. Requires the server to be configured with an HF token (see below).
  3. Start Transcription — click the button and wait. A spinner shows progress; large files can take 30–60 minutes.
  4. Results — once complete you can:
    • Copy the text to clipboard
    • Download as a .txt file (named after your source file)
    • Click Reformat with AI to add speaker labels and clean paragraphs (result appears below the transcription)

✨ Tab 2 — Reformat Text

Use this tab to reformat any existing plain-text transcript — not just files created by this tool.

  1. Either drag & drop a .txt file onto the upload area, or paste text directly into the text box.
  2. Click Reformat with AI.
  3. The AI will assign Speaker 1 / Speaker 2 / … labels based on context, organize the text into clear paragraphs, and lightly clean up repeated filler words.
  4. Copy or download the result. The downloaded file is named after your input file with _formatted appended.

🎤 Speaker Identification Setup

Speaker Identification uses WhisperX and pyannote.audio to analyse the audio waveform and detect who is speaking at each moment. For MP4 uploads, the audio track is extracted automatically first. It is entirely local — no media is sent to any external service.

To enable it:

  1. Create a free Hugging Face account and generate a token at huggingface.co/settings/tokens ↗.
  2. Accept the model licence at pyannote/speaker-diarization-community-1 ↗.
  3. The token is already configured in start.sh — simply restart the server with ./start.sh.
  4. The first run downloads the model (~1 GB, cached permanently after that).

⚠️ Speaker Identification adds roughly 20–50 % to processing time and uses additional GPU memory.

📥 Output File Names

  • Transcription: named after your source file — e.g. interview.mp3 or meeting.mp4 → interview.txt / meeting.txt
  • Formatted output: input name + _formatted — e.g. interview_formatted.txt
